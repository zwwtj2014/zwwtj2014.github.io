在 buildEntry 中则是通过调用 APIrollup.rollup的方式去进行的编译，这里需要注意的有以下几点：

  • 该方法的入参是 input options，不包含 output 配置项
  • 该方法返回一个 promise，resolve 的值可以通过generate生成我们需要的编译之后的代码和 sourcemap
  • generate返回的也是一个 promise，resolve 的是{code, sourcemap}，我们可以在 then 中对其进行处理，比如此处的写文件

整体的原理 CLI 和 API 的方式是一样的，都是通过给配置项编译生成我们需要的源码和 sourcemap
